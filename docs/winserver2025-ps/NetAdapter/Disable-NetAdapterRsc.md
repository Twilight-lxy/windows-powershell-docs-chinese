# 禁用 NetAdapterRsc

## 摘要
禁用网络适配器上的RSC功能。

## 语法

### 按名称排序（默认设置）
```
Disable-NetAdapterRsc [-Name] <String[]> [-IncludeHidden] [-IPv4] [-IPv6] [-NoRestart] [-PassThru]
 [-CimSession <CimSession[]>] [-ThrottleLimit <Int32>] [-AsJob] [-WhatIf] [-Confirm] [<CommonParameters>]
```

### ByInstanceID
```
Disable-NetAdapterRsc -InterfaceDescription <String[]> [-IncludeHidden] [-IPv4] [-IPv6] [-NoRestart]
 [-PassThru] [-CimSession <CimSession[]>] [-ThrottleLimit <Int32>] [-AsJob] [-WhatIf] [-Confirm]
 [<CommonParameters>]
```

### InputObject（cdxml）
```
Disable-NetAdapterRsc -InputObject <CimInstance[]> [-IPv4] [-IPv6] [-NoRestart] [-PassThru]
 [-CimSession <CimSession[]>] [-ThrottleLimit <Int32>] [-AsJob] [-WhatIf] [-Confirm] [<CommonParameters>]
```

## 描述
`Disable-NetAdapterRsc` cmdlet 用于禁用网络适配器上的接收数据段合并（Receive Segment Coalescing，简称 RSC）功能。如果未指定使用 IPv4 还是 IPv6 协议，那么这两种协议都会被同时禁用。RSC 功能会将同一中断期间接收到的多个数据包合并成一个较大的数据包，以便由网络堆栈进行处理。这样做可以降低数据包处理的开销，并减少处理器所需的计算周期数，从而提高系统的可扩展性。

## 示例

#### 示例 1：禁用指定网络适配器上的 IPv6 的 RSC 功能
```
PS C:\> Disable-NetAdapterRsc -Name "MyAdapter" -IPv6
```

这个示例会禁用名为“MyAdapter”的网络适配器上的IPv6协议的RSC（Remote Switching Control）功能，并重新启动该网络适配器。
