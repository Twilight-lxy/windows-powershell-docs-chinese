# 启用 NetAdapter 的 IPSec Offload 功能

## 摘要
启用网络适配器上的IPsec卸载功能。

## 语法

### 按名称排序（默认设置）
```
Enable-NetAdapterIPsecOffload [-Name] <String[]> [-IncludeHidden] [-NoRestart] [-PassThru]
 [-CimSession <CimSession[]>] [-ThrottleLimit <Int32>] [-AsJob] [-WhatIf] [-Confirm] [<CommonParameters>]
```

### ByInstanceID
```
Enable-NetAdapterIPsecOffload -InterfaceDescription <String[]> [-IncludeHidden] [-NoRestart] [-PassThru]
 [-CimSession <CimSession[]>] [-ThrottleLimit <Int32>] [-AsJob] [-WhatIf] [-Confirm] [<CommonParameters>]
```

### InputObject（cdxml）
```
Enable-NetAdapterIPsecOffload -InputObject <CimInstance[]> [-NoRestart] [-PassThru]
 [-CimSession <CimSession[]>] [-ThrottleLimit <Int32>] [-AsJob] [-WhatIf] [-Confirm] [<CommonParameters>]
```

## 描述
`Enable-NetAdapterIPsecOffload` cmdlet 可以在 `AuthHeaderEspEnabled` 状态下启用网络适配器的 IPsec 负载卸载功能。此时，网络适配器会针对每个数据包执行相应的加密操作。这样能够降低处理器的使用率，并可能提高网络吞吐量。在启用负载卸载之前，还需要在 IPsec 策略中配置网络适配器所支持的 IPsec 加密算法。

## 示例

### 示例 1：在指定的网络适配器上启用 IPsec 负载卸载功能
```
PS C:\> Enable-NetAdapterIPsecOffload -Name "MyAdapter"
```

此示例可以在名为“MyAdapter”的网络适配器上启用IPsec卸载功能，并重新启动该网络适配器。
