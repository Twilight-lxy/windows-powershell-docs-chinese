# 禁用 NetQosFlowControl

## 摘要
根据IEEE 802.1p优先级来禁用链路级别的流量控制功能。

## 语法

### 由 IfAlias 提供（默认值）
```
Disable-NetQosFlowControl [[-Priority] <Byte[]>] [[-InterfaceAlias] <String>] [-PassThru]
 [-CimSession <CimSession[]>] [-ThrottleLimit <Int32>] [-AsJob] [-WhatIf] [-Confirm] [<CommonParameters>]
```

### ByIfIndex
```
Disable-NetQosFlowControl [[-Priority] <Byte[]>] [[-InterfaceIndex] <UInt32>] [-PassThru]
 [-CimSession <CimSession[]>] [-ThrottleLimit <Int32>] [-AsJob] [-WhatIf] [-Confirm] [<CommonParameters>]
```

### InputObject (cdxml)
```
Disable-NetQosFlowControl -InputObject <CimInstance[]> [-PassThru] [-CimSession <CimSession[]>]
 [-ThrottleLimit <Int32>] [-AsJob] [-WhatIf] [-Confirm] [<CommonParameters>]
```

## 描述
`Disable-NetQosFlowControl` cmdlet 可以在 Windows Server® 2012 及更高版本中禁用基于优先级的流量控制（PFC）。PFC 是 IEEE 数据中心桥接（DCB）标准的一部分。如果将 Windows Server 2012 或更高版本配置为“不接受来自远程设备的配置”，那么该系统会使用支持 DCB 的网络适配器，从而在链路层禁用 PFC 功能。

有关远程设备配置的更多信息，请参阅 **Set-NetQosDcbxSetting** cmdlet。

有关 PFC 的更多信息，请参阅 **Enable-NetQosFlowControl** cmdlet。

## 示例

#### 示例 1：禁用优先级流量的流量控制
```
PS C:\> Disable-NetQosFlowControl -Priority 6
```

此命令会禁用对标记为优先级 6 的流量进行的流控制（flow control）。
