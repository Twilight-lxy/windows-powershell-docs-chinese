# 撤销 ADAuthenticationPolicySiloAccess 的访问权限

## 摘要
撤销指定账户在某个认证策略（authentication policy）中的成员身份。

## 语法

```
Revoke-ADAuthenticationPolicySiloAccess [-WhatIf] [-Confirm] [-Account] <ADAccount> [-AuthType <ADAuthType>]
 [-Credential <PSCredential>] [-Identity] <ADAuthenticationPolicySilo> [-PassThru] [-Server <String>]
 [<CommonParameters>]
```

## 描述
`Revoke-ADAuthenticationPolicySiloAccess` cmdlet 用于取消一个或多个 Active Directory® Domain Services 中的账户在认证策略隔离（authentication policy silo）中的成员资格。

`Identity` 参数用于指定包含要删除的用户账户的 Active Directory 域服务（AD DS）身份验证策略存储库。您可以通过该存储库的唯一名称、GUID 或名称来识别它。此外，您还可以使用 `Identity` 参数来指定一个包含身份验证策略存储库对象的变量，或者通过管道运算符将身份验证策略对象传递给 `Identity` 参数。

`Account` 参数用于指定要从由 `Identity` 参数指定的身份验证策略集中移除的用户、计算机和服务账户。您可以通过用户的唯一名称（Distinguished Name）、GUID、安全标识符（Security Identifier，SID）或安全帐户管理器（Security Accounts Manager，SAM）账户名来识别相应的用户、计算机或服务账户。此外，您还可以使用 `Account` 参数来指定一个包含用户、计算机和服务账户对象的变量。

## 示例

### 示例 1：撤销对某个身份验证策略（Authentication Policy）的访问权限
```
PS C:\> Revoke-ADAuthenticationPolicySiloAccess -Identity AuthenticationPolicySilo01 -Account User01 -Confirm:$False
```

此命令会撤销用户账户“User01”对名为“AuthenticationPolicySilo01”的身份验证策略存储库的访问权限。由于“Confirm”参数被设置为$False，因此不会显示任何确认消息。

### 示例 2：撤销对用于过滤匹配的认证策略的访问权限
```
PS C:\> Get-ADComputer -Filter 'Name -like "newComputer*"' | Revoke-ADAuthenticationPolicySiloAccess -Identity AuthenticationPolicySilo02
Confirm
Are you sure you want to perform this action?
Performing the operation "Set" on target "CN=Silo,CN=AuthN Silos,CN=AuthN PolicyConfiguration,CN=Services,CN=Configuration,DC=DC01,DC=Contoso,DC=com".
[Y] Yes  [A] Yes to All  [N] No  [L] No to All  [S] Suspend  [?] Help (default is "Y"): A
```

这个命令首先使用 **Get-ADComputer** cmdlet 来获取符合由 `Filter` 参数指定的过滤条件的计算机列表。然后，该输出结果会被传递给 **Revoke-ADAuthenticationPolicySiloAccess** 命令，以移除对名为 `AuthenticationPolicySilo02` 的身份验证策略存储库的访问权限。由于没有指定 `Confirm` 参数，因此会显示一条确认消息。

### -Account
指定要从身份验证策略集中删除的账户。请使用以下格式之一来指定该账户：

- A distinguished name
- GUID
- security identifier
- SAM account name

The cmdlet searches the default naming context or partition to find the object.
If two or more objects are found, the cmdlet returns a non-terminating error.

You can also use this parameter to specify a variable that contains user, computer, and service account objects.
