# 设置 AD 账户身份验证策略（Set-ADAccountAuthenticationPolicySilo）

## 摘要
修改某个账户的身份验证策略或身份验证策略设置（即与该账户相关的身份验证规则）。

## 语法

```
Set-ADAccountAuthenticationPolicySilo [-WhatIf] [-Confirm] [-AuthenticationPolicy <ADAuthenticationPolicy>]
 [-AuthenticationPolicySilo <ADAuthenticationPolicySilo>] [-AuthType <ADAuthType>] [-Credential <PSCredential>]
 [-Identity] <ADAccount> [-PassThru] [-Server <String>] [<CommonParameters>]
```

## 描述
`Set-ADAccountAuthenticationPolicySilo` cmdlet 用于修改某个账户的认证策略或认证策略组（authentication policy silo）。该 cmdlet 会将认证策略组对象（authentication policy silo objects）以及相应的认证策略对象（authentication policy objects）分配给 Active Directory 域服务账户。为了让该账户能够属于某个认证策略组，您需要使用 `Grant-ADAuthenticationPolicySiloAccess` cmdlet 来授予其访问这些对象的权限。

`Identity` 参数用于指定要修改的 Active Directory 域服务（AD DS）身份验证策略。您可以通过该策略的唯一名称、GUID 或名称来识别它。此外，您还可以使用 `Identity` 参数来指定一个包含身份验证策略对象的变量，或者利用管道运算符将身份验证策略对象直接传递给该参数。

## 示例

### 示例 1：分配一个认证策略库（authentication policy silo）和一个具体的认证策略
```
PS C:\> Set-ADAccountAuthenticationPolicySilo -Identity User01 -AuthenticationPolicySilo AuthenticationPolicySilo01 -AuthenticationPolicy AuthenticationPolicy01
```

这个示例将名为 AuthenticationPolicySilo01 的认证策略以及名为 AuthenticationPolicy01 的认证策略分配给名为 User01 的用户账户。

#### 示例 2：通过使用过滤器来分配不同的身份验证策略  

在这个示例中，我们使用了两个不同的身份验证策略（`auth_policy_silo` 和 `auth_policy`），并分别将它们应用于不同的用户或系统组件。具体来说：
```
PS C:\> Get-ADComputer -Filter 'Name -like "NewComputer*"' | Set-ADAccountAuthenticationPolicySilo -AuthenticationPolicySilo AuthenticationPolicySilo02 -AuthenticationPolicy AuthenticationPolicy02
```

这个示例首先使用 **Get-ADComputer** cmdlet 来获取所有符合由 `Filter` 参数指定的过滤条件的计算机账户。该命令的输出结果会被传递给 **Set-ADAccountAuthenticationPolicySilo**，以便将这些计算机账户分配给名为 `AuthenticationPolicySilo02` 的身份验证策略以及名为 `AuthenticationPolicy02` 的另一项身份验证策略。
